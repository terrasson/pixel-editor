## Project Overview

A 32x32 pixel art editor with multi-frame animation support, cloud storage via Supabase, and responsive design for mobile/tablet/desktop. The app allows users to create pixel art animations, save projects to the cloud, and export as GIF files.

## Development Commands

```bash
# Install dependencies (Node.js 22.x required)
npm install

# Start development server (http://localhost:3000)
npm start
# or
npm run dev

# Build (currently placeholder)
npm run build

# Test Supabase GIF edge function
node test-supabase-gif.js

# Test database connection
node test-database-connection.js
```

## Architecture

### Backend (server.js)
- Express server serving static files from `public/` directory
- API endpoints for project persistence: `/api/save`, `/api/load/:filename`, `/api/projects`, `/api/delete/:filename`
- Dual storage strategy:
  - **Local development**: Filesystem storage in `pixel-art-saves/` directory
  - **Production (Vercel)**: In-memory temporary storage (session-only)
- Environment detection via `process.env.NODE_ENV` or `process.env.VERCEL`

### Frontend Architecture

**Core Application (public/js/script.js)**
- Main application state: `frames`, `currentFrame`, `currentColor`, `isDrawing`, `isErasing`
- 32x32 pixel grid (GRID_SIZE constant)
- Frame management with modification tracking via `modifiedPixels` Set per frame
- Animation system with configurable FPS (default: 24)
- Clipboard support for frame copy/paste
- Custom color palette (max 8 colors)
- Auto-save to localStorage with project versioning (max 10 projects)

**Authentication (public/js/auth.js)**
- Supabase-based authentication service
- Session management with automatic state change listeners
- Configuration loaded from `public/js/config/supabase-config.js`

**Database (public/js/database.js)**
- Cloud project persistence to Supabase
- Operations: saveProject, getAllProjects, deleteProject, getProjectByName
- Stores: frames, currentFrame, fps, customPalette, customColors, thumbnail
- Projects are user-scoped (requires authentication)

**User Profile System**
- Profile collection modal with role, experience, and usage tracking
- Local storage keys: `PROFILE_SKIP_KEY`, `PROFILE_PROMPT_DISMISSED_KEY`
- Profile data stored in Supabase when authenticated

### Styling
- Responsive design with separate stylesheets:
  - `styles/common.css` - Shared styles
  - `styles/mobile.css` - Applied on screens ≤1023px
  - `styles/desktop.css` - Applied on screens ≥1024px
  - `styles/login.css` - Login page specific

### Deployment
- Hosted on Vercel
- Configuration in `vercel.json` with route rewrites
- API endpoints proxied through Vercel serverless functions
- Supabase Edge Functions for GIF generation in `supabase/functions/create-gif/`

## Data Structures

### Pixel Format
```javascript
{
  color: '#FFFFFF',  // Hex color string
  isEmpty: true      // Boolean flag
}
```

### Frame Structure
Array of pixel objects (length: GRID_SIZE * GRID_SIZE = 1024)

### Project Data
```javascript
{
  name: string,
  frames: Array<Array<Pixel>>,
  currentFrame: number,
  fps: number,
  customPalette: Array<string>,
  customColors: Array<string>,
  thumbnail: string,  // Generated by database service
  dateCreated: ISO string,
  lastModified: ISO string
}
```

## Key Functions

### Frame Management
- `createEmptyFrame(width, height)` - Creates new blank frame
- `normalisePixel(pixel)` - Validates and normalizes pixel data
- `normaliseFrames(rawFrames)` - Handles various frame data formats from imports

### Supabase Integration
- Authentication requires valid credentials in `public/js/config/supabase-config.js`
- Database schema should be initialized via SQL in Supabase dashboard (see QUICK-START.md)
- Services available globally: `window.authService`, `window.dbService`

## Configuration Files

- `package.json` - Node dependencies: express, @supabase/supabase-js, sharp
- `vercel.json` - Deployment routing configuration
- `.env` - Local environment variables (not tracked in git)
- `public/js/config/supabase-config.js` - Supabase credentials (update with actual values)

## Important Development Notes

1. **Supabase Setup Required**: The app needs Supabase credentials configured before authentication/cloud features work. See QUICK-START.md for setup instructions.

2. **Storage Strategy**: Local development uses filesystem, production uses temporary in-memory storage. For persistent cloud storage, all saves go through Supabase database service.

3. **Node Version**: Requires Node.js 22.x (specified in package.json engines)

4. **Frame Normalization**: Always use `normaliseFrames()` when loading external project data to handle various JSON formats and ensure data integrity.

5. **Responsive Design**: UI adapts based on screen width breakpoint at 1024px. Test both mobile and desktop layouts when making UI changes.

6. **Animation System**: FPS is configurable but defaults to 24. Animation preview managed by `playToggle` button.

7. **Security**: Supabase anon key is safe for client-side code but should ideally use environment variables in production.

## Coding Conventions

- Four-space indentation
- Use `const`/`let`, avoid `var`
- Async/await for I/O operations
- Try/catch error handling in API routes
- Kebab-case for filenames
- Group DOM queries near event listeners
- Commit messages: lowercase imperative or Conventional Commits format (feat:, fix:)
